# def_manager
## Проект для управления документами, полученными с портала КадАрбитр
#### Вместо telegram-бота
### Цель:
- Парсинг данных о движении документов на портале
- Парсинг дат судов
- Отображение движения дел в ЛК
- Управление документами клиентов (автоматическое создание запросов на основе полученных данных с портала)

### Используемые технологии
- FastAPI
- Postgresql
- Playwright
- SQLAlchemy
- Celery или (dramatiq)
